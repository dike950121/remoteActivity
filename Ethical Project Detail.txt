Project: Secure Remote Administration Tool (SecureRAT)

Purpose:
Developing a legitimate, transparent, and secure remote administration tool for authorized system administrators to manage corporate networks with proper consent and security controls.

Key Ethical Requirements:

1. Transparency
- All processes must be visible to users and system administrators
- Clear documentation of all features and capabilities
- Open source code for security auditing
- Visible system tray icon when running

2. Security Controls
- Mandatory authentication with strong password requirements
- Multi-factor authentication support
- Encrypted communications using industry-standard protocols (TLS 1.3)
- Complete audit logging of all actions
- Session recording for compliance

3. User Consent and Privacy
- Explicit user consent required before installation
- Clear notifications when remote access is active
- Option for users to terminate remote sessions
- No hidden or background operations without user awareness
- Compliance with GDPR and privacy regulations

4. Installation and Management
- Standard installer package with clear documentation
- Proper digital code signing
- Installation requires administrative privileges
- Easy uninstallation process
- Regular security updates and patches

5. Features (All requiring explicit authorization)
- Remote desktop assistance with user approval
- Secure file transfer with encryption
- System health monitoring
- Software deployment tools
- Remote PowerShell with logging

6. Security Best Practices
- Regular security audits
- Vulnerability scanning
- Input validation and sanitization
- Protection against common attack vectors
- Rate limiting to prevent abuse

7. Compliance Requirements
- SOC 2 compliance
- GDPR compliance
- HIPAA compliance (if used in healthcare)
- Industry-standard security certifications

Technical Requirements:
- Built with C#/.NET for Windows environments
- Clean, maintainable, and well-documented code
- Proper error handling and logging
- Unit and integration tests
- Performance optimization

The ideal developer should have:
- Strong background in secure software development
- Experience with Windows application development
- Knowledge of network security protocols
- Understanding of compliance requirements
- Commitment to ethical coding practices
